## 5. Findings (new this run)

- **Low — `.at` (Austrian) source surfaced via Tier-4, extracted 2 deals, mislabeled `country: DE`.** `magenta.at/tv/disney-plus` produced 2 deals stamped `country: DE` (the active market is hard-coded DE). Both were must-review *this* run (better than the 2026-06-20 `.ch` Swisscom deal that passed the gate), but the `currency_matches_country` rule **cannot** catch it because **Austria also uses EUR** — only a source-ccTLD/country check would. **This is the existing KNOWN_ISSUES "Tier-4 can surface a non-DE source" entry, now with a sharper second data point (EUR-ccTLD case the currency guard can't catch).** Where: `src/domain/discovery/domain-denylist.ts`, `src/domain/deal-record/enums.ts`. **Defer** — updated the existing KNOWN_ISSUES entry rather than logging a new one. Fix-when: Tier-4 curation / multi-country (add non-DE ccTLDs to the deny-list and/or a source-ccTLD sanity rule).
- **Low (curation, not code) — the mydealz seed RSS URL (`mydealz.de/rss/alle`) is dead (HTTP 404).** `parseFeed` handled it correctly (0 items, no crash). The seed list (`docs/DealRoute_Seed_List_DE.md`) should be refreshed with a live mydealz feed/search endpoint. **Defer** — logged to KNOWN_ISSUES (seed curation).
- **Info — per-source fetcher selection matters for JS-heavy SPAs.** ChatGPT pricing (650k chars, schema-fail→re-ask) and NordVPN `/de/` (0 deals on both playwright and browser-render) confirm that a service's *landing* URL is often not its *pricing* URL, and SPA pricing may need a deeper/path-specific URL. Not a bug — both produced honest 0/low-confidence results, no hallucination. Reinforces the per-source URL-curation + `FETCHER=browser` selection already noted in prior runs.
- **Info — all hardening built since the last live run fired correctly in the wild:** bounded re-ask **recovered** 3 times (ChatGPT, O2, Telekom-Disney); input-size cap trimmed 4 real pages (650k/364k/161k/132k) with no crash; `extraction_input_truncated` flagged every trimmed deal to must-review; a Firecrawl 30s search timeout was caught and the lane continued at €0.

**No blocker or high-severity findings. No trust-invariant breach. No crashes. 0 auto-publishes.**

## 6. Expectations vs reality

The run matched expectations and confirmed the pipeline is solid in the wild at the
post-Step-6 feature-complete state. Every tier produced trustworthy, evidence-backed,
must-review candidates; the two deals that passed the gate (Spotify Duo/Family) are
exactly the kind of clean, fully-grounded standalone prices that *should* pass. All seven
trust invariants held: Netflix was politely refused on robots; the Tier-4 inline-scrape
content was only used after the robots gate; nothing auto-published and nothing was
written in any dry-run; unknown conditions became field_proposals, never columns; the
hallucination guard caught ungrounded validity quotes. The hardening shipped since the
last live run all fired correctly — most satisfyingly the **bounded re-ask recovered three
times** and a **Firecrawl search timeout was swallowed without crashing the lane**. The
only finding of note is a repeat of the known Tier-4 non-DE-source issue, now sharpened: a
`.at` EUR source slips past the currency guard (because AT uses EUR) and is mislabeled DE —
a curation/multi-country matter, not a trust breach, since nothing auto-publishes and a
human approves the proposed domain. Next: refresh the mydealz seed feed URL; when curating
Tier-4 / enabling a 2nd country, add non-DE ccTLDs to the deny-list and a source-country
sanity rule.
